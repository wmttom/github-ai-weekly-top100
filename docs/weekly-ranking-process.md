# GitHub AI 周榜生成流程说明

本文沉淀当前 `wmttom/github-ai-weekly-top100` 项目的周榜生成规则、数据源口径、排序逻辑、输出格式与运行注意事项。

## 1. 目标

每周生成一份 `GitHub Top100 AI 项目趋势榜`，重点体现 **本周趋势变化**，而不是累计总 stars 排行。

输出结果保存为 CSV 文件，写入本仓库，文件名格式：

- `YYYY年MM月DD日.csv`

例如：

- `2026年04月17日.csv`

## 2. 数据源优先级

周榜生成时优先参考以下信号：

1. GitHub Trending（weekly）
2. GitHub Trending（当前 / daily）
3. 用户补充的趋势数据（如 today stars / weekly stars）
4. GitHub 仓库搜索结果（作为补充候选池）
5. 本仓库上周榜单 CSV（用于排名变化对比）

## 3. 候选项目筛选口径

候选项目应满足以下原则：

- 属于 AI 相关项目
- 尽量覆盖 coding、agent、memory、MCP、browser、RAG、voice、video、finance、education 等方向
- 重点体现本周新增热度
- **不纳入 1 年以上的老项目**
- 若是老项目，则只有在上周已上榜且本周仍有明显趋势信号时才保留

### 应优先纳入的高趋势项目特征

- 本周 stars 增长非常高
- 今日 stars 增长也很高
- 同时出现在 weekly / daily 趋势里
- 属于本周明显升温的 AI 方向

### 应下调或剔除的项目特征

- 主要依赖历史累计 stars，但本周增量弱
- 只是资源合集、技能清单、教程，但缺乏本周新增热度
- 超过 1 年且未体现明显趋势变化

## 4. 重点跟踪的 AI 方向

当前周榜生成时，优先关注以下方向：

- AI coding / Claude Code / coding agent
- 通用 Agent / 多代理协作
- Agent memory / 长上下文
- MCP / 工具协议 / 浏览器工具链
- Browser agent / Web automation
- RAG / 文档解析 / 知识库
- Voice / TTS / STT
- 金融 AI / 量化 agent
- 教育类 agent
- 视频生成 / 自动剪辑

## 5. 排序思路

榜单不是累计 stars 榜，而是 **趋势榜**。

排序时优先考虑：

1. 本周 stars 增长
2. 今日 stars 增长
3. 是否属于最近一年内项目
4. 是否已在上周榜单中
5. 是否属于本周明显升温的 AI 子方向

## 6. 对比上周排名变动

必须读取本仓库上周 CSV 文件，计算 `对比上周排名变动` 字段。

规则如下：

- 上周无该项目：`新上榜`
- 本周排名优于上周：`上升N`
- 本周排名低于上周：`下降N`
- 本周与上周相同：`持平`

上周文件使用同一仓库内上一个周五对应的 CSV 作为基线。

## 7. 输出字段

CSV 表头固定为：

```csv
排名,项目,热度分,对比上周排名变动,项目分类,项目 License,项目简介
```

字段要求：

- `排名`：1~100 连续整数
- `项目`：`owner/repo`
- `热度分`：体现本周趋势的代理分值
- `对比上周排名变动`：基于上周文件计算
- `项目分类`：使用固定分类
- `项目 License`：真实 license；无法确认时填 `未知 / 未声明`
- `项目简介`：必须采用固定结构：
  - `场景：...` 
  - `能解决什么问题：...`
  - `适合做什么：...`

## 8. 项目简介写法

简介统一写成：

- 场景：该项目适合什么场景
- 能解决什么问题：它解决的核心痛点是什么
- 适合做什么：适合落地在哪些任务或业务里

示例：

```text
场景：AI 编码与工程协作。能解决什么问题：让代码生成、修改、测试和规范沉淀更稳定，减少提示词随机性和工程流程失控。适合做什么：日常开发、重构、测试补全和团队研发提效。
```

## 9. GitHub 写入规则

生成完成后：

1. 以东八区日期生成文件名
2. 写入本仓库根目录
3. 若同名文件已存在，则更新覆盖
4. 返回简短执行结果：
   - 文件名
   - 是否找到上周文件
   - 本周是否存在大量新上榜项目

## 10. 当前周榜趋势总结口径

周榜总结时，优先从以下几个方面概括：

- 本周最热的 AI 主方向
- coding / agent / memory / MCP / voice / finance / education 等哪个方向在升温
- 哪类旧热点在回落
- 哪些项目体现了强烈的本周趋势变化

## 11. 注意事项

- 周榜应突出“本周变化”，不是累计榜
- 不要把 1 年以上的老项目简单拉进前排
- 对于用户明确补充的 today / weekly 趋势数据，优先级应提高
- 在正式写入前，先确保榜单项目唯一、排名连续、字段完整
