生成当前周 GitHub Top100 AI 项目趋势榜，并将结果以 CSV 文件写入仓库 `wmttom/github-ai-weekly-top100`。

执行要求：

1. 使用 Asia/Shanghai 时区。
2. 目标文件名格式必须为：`YYYY年MM月DD日.csv`
3. 结果必须写入 GitHub 仓库根目录；若同名文件已存在，则更新覆盖。
4. 输出 CSV 表头固定为：
   `排名,项目,热度分,对比上周排名变动,项目分类,项目 License,项目简介`

5. 榜单目标是体现 **本周趋势变化**，不是累计总 stars 排名。
6. 优先参考以下信号：
   - GitHub Trending weekly
   - GitHub Trending 当前/日榜
   - 今日 stars 增量
   - 本周 stars 增量
   - 用户补充的趋势数据
   - GitHub 仓库搜索候选
   - 上周榜单 CSV

7. 候选项目筛选规则：
   - 必须是 AI 相关项目
   - 尽量覆盖 coding、agent、memory、MCP、browser、RAG、voice、video、finance、education 等方向
   - **不纳入 1 年以上的老项目**
   - 若项目超过 1 年，只有在上周已上榜且本周仍有明显趋势信号时才保留

8. 排序时优先考虑：
   - 本周 stars 增长
   - 今日 stars 增长
   - 是否属于最近一年项目
   - 是否已在上周榜单中
   - 是否属于本周明显升温的 AI 子方向

9. 必须读取本仓库中上周对应的 CSV 文件，并计算 `对比上周排名变动`：
   - 上周无该项目：`新上榜`
   - 本周更靠前：`上升N`
   - 本周更靠后：`下降N`
   - 相同：`持平`

10. `项目简介` 必须统一写成下面的结构：
   - `场景：...`
   - `能解决什么问题：...`
   - `适合做什么：...`

11. `项目 License` 尽量写真实值；无法确认时填：`未知 / 未声明`

12. 必须保证：
   - Top100 项目唯一
   - 排名连续 1~100
   - 字段齐全
   - CSV 格式合法

13. 完成后返回：
   - 生成的文件名
   - 是否找到上周文件
   - 本周新增纳入的重点项目
   - 本周显著上升和显著下降的项目方向
